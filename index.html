Домашнє завдання: GIT

Base

  • git config --global user.name blabla - задати параметр (ім'я) для даного проекту
  • git config --global user.email blabal@bla -задати параметр (емеіл) для даного проеку
  • git init - ініціалізація нового репозиторія
  • git status - перевірка стану репозиторія
  • git add - добавити файл в індекс(staging area) для комміту
  • git commit - переміщення даних зі staging area і зберігає їх зліпок у внутрішній базі даних
  • git log- показує історію коммітів
  • git checkout - переключення коммітів, переключення гілок та вивантаження вмісту в робочу директорію
  • git branch - "менеджер гілок -"управління гілками (створення, об'єднання, видалення гілок), команда задана без параметрів показує наявних гілок
  • git push - інформацію з локального репозиторія вигрузити у віддалений репозиторій
  • git pull - стягнути зміни з віддаленого на локальний репозиторій
  • git merge - використовується для злиття однієї або кілької гілок в поточну гілку
  • git clone - клонувати віддалений репозиторій на локальний
  • Файл .gitignore - файл(інструмент), в якому розміщуються файли приховані для git.

Advanced

  • git revert - скасування комміту
  • git reset - відмінити індексацію даних
  • git rebase - перезаписує історію коммітів
  • git cherry-pick - повторне використання комміту в іншій гілці
  • git stash - тимчасове збереження всіх незакоммічених змін для очистки робочої директорії
